# Как привязать схему посадочных мест к автобусу

## Обзор
Функционал привязки схем посадочных мест к автобусам уже реализован и работает. Вот подробная инструкция.

## Шаг 1: Создание схемы посадочных мест

1. Перейдите в админ-панель: `http://127.0.0.1:8000/admin/`
2. В разделе "Tickets" выберите "Seat layouts"
3. Нажмите "ADD SEAT LAYOUT"
4. Заполните поля:
   - **Name**: Название схемы (например, "Стандартная 2+2")
   - **Description**: Описание (необязательно)
   - **Total seats**: Общее количество мест
   - **Seats per row**: Мест в ряду
5. Нажмите "SAVE"

## Шаг 2: Привязка схемы к автобусу

1. В админ-панели перейдите в "Tickets" → "Buses"
2. Выберите автобус для редактирования
3. В поле "Seat layout" выберите нужную схему из выпадающего списка
4. Нажмите "SAVE"

## Текущие схемы в системе

Сейчас доступны следующие схемы:

### 1. Standard 2+2 (40 мест)
- **Описание**: Стандартная схема для обычных автобусов
- **Конфигурация**: 2+2 места в ряду
- **Всего мест**: 40
- **Рядов**: 10

### 2. Minibus 2+3 (18 мест)
- **Описание**: Схема для микроавтобусов
- **Конфигурация**: 2+3 места в ряду
- **Всего мест**: 18
- **Рядов**: 4

### 3. Luxury 2+1 (24 места)
- **Описание**: Люкс схема с повышенным комфортом
- **Конфигурация**: 2+1 места в ряду
- **Всего мест**: 24
- **Рядов**: 8

### 4. Double Decker 4+4 (72 места)
- **Описание**: Схема для двухэтажных автобусов
- **Конфигурация**: 4+4 места в ряду
- **Всего мест**: 72
- **Рядов**: 9

### 5. Test Layout (12 мест)
- **Описание**: Тестовая схема
- **Конфигурация**: 3 места в ряду
- **Всего мест**: 12
- **Рядов**: 4

## Текущее назначение схем

- **BUS001**: Standard 2+2
- **BUS002**: Standard 2+2
- **BUS003**: Luxury 2+1

## Как изменить схему у автобуса

1. Откройте автобус в админ-панели
2. В поле "Seat layout" выберите другую схему
3. Нажмите "SAVE"
4. Новая схема автоматически применится ко всем рейсам этого автобуса

## Визуализация схем

В админ-панели вы можете видеть визуализацию схем:
- В списке схем - миниатюрный preview
- При редактировании схемы - полная визуализация
- При редактировании автобуса - preview назначенной схемы

## Автоматическая генерация

Если у схемы не указаны JSON данные, система автоматически генерирует схему на основе параметров:
- Распределяет места по рядам
- Нумерует места от 1 до total_seats
- Создает правильную структуру для отображения

## Ручное создание схем

Для сложных конфигураций можно указать JSON данные вручную:

```json
[
  [1, 2, null, 3, 4],
  [5, 6, null, 7, 8],
  [9, 10, null, 11, 12]
]
```

Где:
- Числа: номера мест
- `null`: пустое пространство (проход, дверь)

## Проверка функционала

Запустите тестовый скрипт для проверки:
```bash
python check_seat_layout_assignment.py
```

## Преимущества системы

1. **Гибкость**: Любой автобус может использовать любую схему
2. **Визуализация**: Схемы наглядно отображаются в админ-панели
3. **Автоматизация**: Легко создавать новые схемы
4. **Совместимость**: Работает с существующими рейсами
5. **Масштабируемость**: Легко добавлять новые типы автобусов
