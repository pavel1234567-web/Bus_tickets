# Руководство по управлению схемами посадочных мест

## Обзор

Добавлен новый функционал для управления схемами посадочных мест автобусов в админ-панели.

## Новые возможности

### 1. Модель SeatLayout
- **Назначение**: Хранение конфигураций схем посадочных мест
- **Поля**:
  - `name`: Название схемы
  - `description`: Описание схемы
  - `total_seats`: Общее количество мест
  - `seats_per_row`: Количество мест в ряду
  - `layout_data`: JSON данные схемы (опционально)
  - `is_active`: Активность схемы

### 2. Обновленная модель Bus
- Добавлено поле `seat_layout_config` для связи со схемой посадочных мест
- Сохранена обратная совместимость с существующими автобусами

### 3. Админ-интерфейс

#### Управление схемами посадочных мест
- **URL**: `/admin/tickets/seatlayout/`
- **Функции**:
  - Создание новых схем
  - Редактирование существующих схем
  - Визуализация схем в виде таблицы
  - Автоматическая генерация схем на основе параметров

#### Управление автобусами
- **URL**: `/admin/tickets/bus/`
- **Новые функции**:
  - Выбор схемы посадочных мест из выпадающего списка
  - Предпросмотр схемы мест для каждого автобуса
  - Фильтрация по схемам посадочных мест

## Использование

### Создание новой схемы посадочных мест

1. Перейдите в админ-панель: `/admin/tickets/seatlayout/`
2. Нажмите "Add Seat Layout"
3. Заполните основные поля:
   - **Name**: Название схемы (например, "Standard 2+2")
   - **Description**: Описание (например, "Стандартная схема для обычных автобусов")
   - **Total seats**: Общее количество мест
   - **Seats per row**: Количество мест в ряду
4. Нажмите "Save"
5. Система автоматически сгенерирует схему на основе указанных параметров

### Привязка схемы к автобусу

1. Перейдите в админ-панель: `/admin/tickets/bus/`
2. Выберите автобус для редактирования
3. В поле "Seat layout" выберите нужную схему из выпадающего списка
4. Нажмите "Save"

### Ручное создание схемы (опционально)

Если стандартная генерация не подходит, можно вручную указать JSON данные в поле "Layout data":

```json
[
  [1, 2, null, 3, 4],
  [5, 6, null, 7, 8],
  [9, 10, null, 11, 12]
]
```

Где:
- Числа: номера мест
- `null`: пустое пространство (проход, дверь и т.д.)

4. **Double Decker 4+4** (72 места) - схема для двухэтажных автобусов

## Технические детали

### Методы модели SeatLayout

- `get_layout_display()`: Возвращает данные схемы для отображения
- `generate_default_layout()`: Генерирует стандартную схему на основе параметров

### Обратная совместимость

Автобусы без привязанной схемы продолжают работать с автоматически генерируемой схемой на основе `total_seats` и `seats_per_row`.

### Миграции базы данных

Выполнены следующие миграции:
- `0007_seatlayout_bus_seat_layout_config.py`: Создание модели SeatLayout и добавление поля в Bus
- `0008_alter_seatlayout_layout_data.py`: Изменение поля layout_data для поддержки NULL значений

## Преимущества

1. **Гибкость**: Возможность создания различных схем для разных типов автобусов
2. **Визуализация**: Наглядное отображение схем в админ-панели
3. **Автоматизация**: Автоматическая генерация схем по параметрам
4. **Совместимость**: Полная обратная совместимость с существующими данными
5. **Масштабируемость**: Легкое добавление новых схем по мере необходимости
